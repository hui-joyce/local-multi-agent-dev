### Assumptions
1.  **Concurrency**: The implementation assumes a multi-threaded environment. Therefore, the `TokenBucket` class uses a `threading.Lock` to ensure thread safety when acquiring tokens.
2.  **Time Source**: The implementation uses `time.monotonic()` for timing. This is preferred over `time.time()` for rate limiting as it is not affected by system clock adjustments (e.g., NTP synchronization), preventing race conditions in token refill calculations.
3.  **Token Precision**: Tokens are stored as floats to allow for fractional accumulation (e.g., 0.5 tokens), which provides smoother rate limiting than integer-only buckets.
4.  **Usage Pattern**: The limiter is designed to be instantiated once per resource (e.g., one limiter for API calls, another for database writes) and reused across threads.

```python
import threading
import time
from typing import Optional, Tuple

class TokenBucket:
    """
    A thread-safe Token Bucket rate limiter.
    
    Logic:
    - The bucket has a maximum capacity (max_tokens).
    - Tokens are added at a constant rate (refill_rate) per second.
    - Requests consume tokens. If tokens are available, the request is allowed.
    - If tokens are exhausted, the request must wait until enough tokens are refilled.
    
    Thread Safety:
    - Uses a threading.Lock to protect the internal state during acquire operations.
    """

    def __init__(self, max_tokens: float, refill_rate: float):
        """
        Initialize the token bucket.
        
        Args:
            max_tokens (float): The maximum number of tokens the bucket can hold.
            refill_rate (float): The number of tokens added per second.
        """
        self.max_tokens = max_tokens
        self.refill_rate = refill_rate
        self.tokens = max_tokens  # Start with a full bucket for burst capability
        self.last_refill_time = time.monotonic()
        self._lock = threading.Lock()

    def _refill(self) -> None:
        """
        Internal method to refill tokens based on elapsed time.
        Must be called while holding the lock.
        """
        now = time.monotonic()
        elapsed = now - self.last_refill_time
        tokens_to_add = elapsed * self.refill_rate
        
        self.tokens = min(self.max_tokens, self.tokens + tokens_to_add)
        self.last_refill_time = now

    def acquire(self, tokens: float = 1.0) -> Tuple[bool, float]:
        """
        Attempt to acquire the specified number of tokens.
        
        Args:
            tokens (float): Number of tokens to acquire. Defaults to 1.0.
            
        Returns:
            Tuple[bool, float]: 
                - True if tokens were acquired.
                - False if not enough tokens are available.
                - The second value is the estimated wait time in seconds if denied.
        """
        with self._lock:
            # Refill tokens based on time passed since last check
            self._refill()

            if self.tokens >= tokens:
                self.tokens -= tokens
                return True, 0.0
            else:
                # Calculate how long we need to wait to get the required tokens
                tokens_needed = tokens - self.tokens
                wait_time = tokens_needed / self.refill_rate
                return False, wait_time

    def wait(self, tokens: float = 1.0) -> None:
        """
        Block the current thread until the specified number of tokens are available.
        This is a blocking version of acquire.
        
        Args:
            tokens (float): Number of tokens to acquire. Defaults to 1.0.
        """
        while True:
            acquired, wait_time = self.acquire(tokens)
            if acquired:
                return
            # Sleep for a small fraction of the wait time to avoid busy-waiting
            # Using a small sleep prevents high CPU usage while waiting
            sleep_time = min(wait_time, 0.01)
            time.sleep(sleep_time)

# Example Usage
if __name__ == "__main__":
    # Allow 10 requests per second, with a burst capacity of 20
    limiter = TokenBucket(max_tokens=20, refill_rate=10)
    
    # Simulate a burst of 25 requests

        if allowed:

        else:
```

### Key Design Decisions Explained
*   **Float vs Int**: Using floats allows for sub-second precision in rate limiting (e.g., 1.5 requests per second), which is often required for API throttling.
*   **`time.monotonic()`**: This ensures that if the system clock is adjusted (e.g., via NTP or manual change), the rate limiter's internal clock does not jump, preventing the bucket from suddenly gaining or losing tokens unexpectedly.
*   **Locking Strategy**: A single `threading.Lock` protects the entire `acquire` logic. While this introduces slight contention, it is the simplest and most robust way to prevent race conditions in Python's GIL-less context for float arithmetic.
*   **Non-blocking `acquire`**: The `acquire` method returns immediately with a boolean and a wait time estimate. This allows the caller to make a decision (e.g., queue the request or reject it) without blocking the thread, which is crucial for high-concurrency servers.
*   **Blocking `wait`**: The `wait` method is provided for scenarios where the caller can afford to block (e.g., a background worker thread) and wants to proceed as soon as a token is available.

### Analysis of the Pseudo-Code

**Likely Intent:**
The code implements a **state-machine based protocol parser** with a custom **streaming checksum** and **obfuscation**.
1.  **State Machine:** It tracks the progression of a data stream through three distinct phases: `INIT` (looking for start), `HEADER` (looking for separator), and `BODY` (processing payload).
2.  **Obfuscation:** The input bytes are XORed with a repeating `key` before processing. This suggests the input is encrypted or encoded.
3.  **Checksum:** A rolling checksum is calculated on the *obfuscated* bytes (multiplied by 17). The final checksum must match `0xBEEF` for the operation to succeed.
4.  **Execution:** If the format is valid and the checksum matches, a `privileged_operation` is triggered.

**Hidden Assumptions:**
1.  **Key Length:** The code assumes `len(key) > 0`. If `key` is empty, `idx % len(key)` will raise an `IndexError`.
2.  **Input Length:** The code assumes `len(input) >= 3`. If the input is shorter than the required markers (`{`, `:`, `}`), the state will never reach `DONE`, resulting in `ERR_FORMAT`.
3.  **Byte Representation:** The pseudo-code treats `input` and `key` as sequences of integers (0-255). In Python, this usually means a `bytes` object or a list of integers.
4.  **State Transitions:** The logic assumes that once a state is entered (e.g., `HEADER`), it stays there until the specific exit condition (`0x7D`) is met. It does *not* handle cases where the input contains multiple markers in a single stream (e.g., `0x7B` appearing again after `0x7D`).
5.  **Checksum Scope:** The checksum accumulates over the *entire* stream, not just the payload. This means the header bytes (`0x7B`, `0x3A`) contribute to the final `0xBEEF` value.

**Possible Abuse Cases:**
1.  **Key Exhaustion/Replay:** If the `key` is short, the XOR pattern repeats. An attacker could potentially find a short key that, when XORed with a specific payload, results in the correct checksum.
2.  **State Injection:** If the input stream is malformed (e.g., missing the `0x3A` but having `0x7D` later), the state machine logic in the pseudo-code is brittle.     HEADER = 1
    BODY = 2

class ErrorCode(Enum):
    ERR_FORMAT = "ERR_FORMAT"
    ERR_AUTH = "ERR_AUTH"
    ERR_KEY_EMPTY = "ERR_KEY_EMPTY"

    Verifies a stream against a state machine and a checksum, then executes a privileged operation.
    
    Args:
        input_data: The raw byte stream to process.
        key: The repeating XOR key.
        privileged_op: A callable to execute if verification passes.

        """

```python
        In production, this would be a database connection or Redis cache.
        """
        self.user_db = user_db
        self._generate_jwt_secret()

    def _generate_jwt_secret(self):
        # Assumption: In a real app, this reads from env vars. 
        # Here we generate a random one for the demo session.
        self.jwt_secret = secrets.token_urlsafe(32)

    def register(self, payload: AuthPayload) -> Dict[str, Any]:
        """
        Registers a new user and returns a JWT token.
        """
        email = payload.email
        password = payload.password
        
        # Check if user exists
        if email in self.user_db:
            return {"error": "User already exists", "status": 400}

        # Hash password
        password_hash = PASSWORD_HASHING_CONTEXT.hash(password)
        
        # Generate unique ID
        user_id = secrets.token_hex(8)
        user = User(
            id=user_id,
            email=email,
            password_hash=password_hash,
            created_at=datetime.utcnow(),
            is_active=True
        )
        
        self.user_db[email] = user
        
        # Generate Token
        token = self._create_token(user)
        
        return {"token": token, "user": user.to_dict(), "status": 201}

    def login(self, payload: AuthPayload) -> Dict[str, Any]:
        """
        Authenticates a user and returns a JWT token.
        """
        email = payload.email
        password = payload.password
        
        if email not in self.user_db:
            return {"error": "Invalid credentials", "status": 401}
        
        user = self.user_db[email]
        
        if not user.is_active:
            return {"error": "Account disabled", "status": 403}
        
        # Verify password (Constant time comparison is handled by bcrypt internally)
        if not PASSWORD_HASHING_CONTEXT.verify(password, user.password_hash):
            return {"error": "Invalid credentials", "status": 401}
            
        token = self._create_token(user)
        return {"token": token, "user": user.to_dict(), "status": 200}

    def decode_token(self, token: str) -> Optional[Dict[str, Any]]:
        """
        Decodes and validates a JWT token.
        """
        try:
            payload = jwt.decode(
                token, 
                self.jwt_secret, 
                algorithms=[JWT_ALGORITHM],
                options={"require": ["exp", "sub"]}
            )
            return payload
        except jwt.ExpiredSignatureError:
            return {"error": "Token expired", "status": 401}
        except jwt.InvalidTokenError:
            return {"error": "Invalid token", "status": 401}

    def _create_token(self, user: User) -> str:
        to_encode = {
            "sub": user.id,
            "email": user.email,
            "role": user.role,
            "exp": datetime.utcnow() + timedelta(minutes=JWT_EXPIRATION_MINUTES)
        }
        return jwt.encode(to_encode, self.jwt_secret, algorithm=JWT_ALGORITHM)
```
